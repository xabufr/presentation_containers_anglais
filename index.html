Containers

A quick technical tour

Containers are like virtual machines...

  • They allow processes to be isolated from each other, in term of
    • Storage
    • Network
    • System access
    • Resources usage
    • Dependencies

...and this really powerfull

  • Very little performance overhead
  • Lightweight images
  • Easy to build

It seems main problems we had with virtualization have gone !

Cool !

But...

Containers can do better...

Much better !

They are, unlike VMs images, immutable
They fill the gap between Developers and Operators

You can easily share then with people.

For instance by using docker and hub.docker.com

And they are actually used by many big companies

Google use them since a decade, with a container orchestration system called Borg.

Wait, a container orchestration system ? What is this ?

Container orchestration system

If I tell you that you don't have to care about servers ?
You just have a cluster. The system automatically launch as many instances of your image you requests, with the possibility to automatically scale it if needed.
If I tell you that it can also automate all your deployment process, with zero downtime for users, with only few lines of configuration files ?
